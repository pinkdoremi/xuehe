深入了解React生命周期

什么是生命周期:

react的生命周期就是react组件从创建,到更新,最后到销毁这段时间.

react在这段时间的不同阶段提供了不同的接口,在经历不同的生命周期中的节点时,会调用这些方法.

通过在不同的时间阶段写入特殊的方法,能够让我们能够很好地控制组件的状态,让组件按照开发者的心意去安排任意的状态,从而渲染出开发者希望的样子,并且可以做任何想做的事情.

一个复合组件的生命周期:

  1. constructor : 此时初始化state,并且创建实例.
  2. componentWillMount : 组件即将被挂载到页面上.
  3. render : 在此方法中,仅根据组件的state以及props进行渲染.
  4. 下面进入子组件的生命周期:
    1. 子组件的:constructor
    2. 子组件的:componentWillMount
    3. 子组件的:render
    4. 子组件的:componentDidMount
    5. 子组件更新阶段生命周期:
      1. componentWillReceiveProps: 只要父组件更新,此方法就会被调用
      2. shouldComponentUpdate: return true/false,来决定是否更新此组件
      3. componentWillUpdate: 组件更新之前最后一环.
      4. render
      5. 孙子组件进行更新或创建…
      6. componentDidUpdate: 组件更新完毕后调用
    6. componentWillUnmount:子组件将被移除时调用,用于解除事件绑定等操作.
    7. 孙子组件的componentWillUnmount
    8. 孙子组件销毁
      …
  5. 此组件销毁,实例为空,被react释放,等待垃圾回收;

强行缩减项目周期对项目造成的危害

最近遇到一个非常紧急并且重要的项目,从产品设计到交互视觉到开发的各个部分的时间都需要缩短.项目最终顺利上线.但是这样的过程,产生了很多问题:

1. 开发周期缩短:

开发周期缩短,但是功能不能砍的情况下,只能靠添加人手来解决这个问题.但是添加临时人手又发生这些问题:

1. 临时的人手对项目技术不熟悉,过程中说明和指导带来的时间成本非常高
2. 临时的人手代码实现方式各有不同,不同的人修改的成本非常高,临时人手如果不能维护到项目上线,这部分代码在测试阶段面对bug和需求变更,会让维护它的老人手非常闹心,容易改出bug,风险也较大.

如果开发过程中思考较多的话,精心设计的代码逻辑和架构也能够提高开发效率,提高开发质量.但是这一点最怕的就是需求变更!
开发周期缩短还会导致开发质量下降,没有足够时间,这太容易发生了.

PhantomJs是什么？

Phantom JS是一个服务器端的 JavaScript API 的 WebKit。支持各种Web标准： DOM 处理, CSS 选择器, JSON, Canvas, 和 SVG。

用来干嘛的

BrowserSync可以监听你的页面文件的变化。每当页面文件变化时，所有访问该站点的设备都会同步刷新，并且显示最新的代码。
此外它还有个超级厉害的功能就是在任意一个设备上的操作，会同步到别的设备上，你下拉，别的页面也会下拉，你点按钮，别的页面也会点按钮。ui验收的时候，摆上一排机器，随便拉拉，页面都在晃，超级拉风的，而且对效率的提升非常高。
